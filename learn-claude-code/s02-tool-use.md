工具使用：

```
    +----------+      +-------+      +------------------+
    |   User   | ---> |  LLM  | ---> | Tool Dispatch    |
    |  prompt  |      |       |      | {                |
    +----------+      +---+---+      |   bash: run_bash |
                          ^          |   read: run_read |
                          |          |   write: run_wr  |
                          +----------+   edit: run_edit |
                          tool_result| }                |
                                     +------------------+
```

扩展工具能力。

**那 LLM 如何知道要调用哪些 tools，有哪些 tools？**

1.  **原生 Tool Calling**：通过接口参数传 `tools`（函数名+schema），模型返回结构化 tool call，应用执行后回传结果
1.  **Prompt 内嵌工具说明（纯文本 ReAct/约定格式）**：在 prompt 中写工具列表和调用格式，模型按文本约定输出“调用指令”

<br/>

| 方案                                   | 核心做法                                                    | 优点                           | 缺点                     | 适用场景             |
| ------------------------------------ | ------------------------------------------------------- | ---------------------------- | ---------------------- | ---------------- |
| 原生 Tool Calling（API/SDK 内置）          | 通过接口参数传 `tools`（函数名+schema），模型返回结构化 tool call，应用执行后回传结果 | 稳定、标准化；参数校验强；易观测与审计；多轮工具链路清晰 | 依赖平台能力；接入初期要理解协议       | 生产系统首选、对可靠性要求高   |
| Prompt 内嵌工具说明（纯文本 ReAct/约定格式）        | 在 prompt 中写工具列表和调用格式，模型按文本约定输出“调用指令”                    | 实现简单；跨模型/平台迁移容易；适合教学原型       | 易格式漂移；解析脆弱；注入风险高；维护成本高 | PoC、教学、无原生工具协议时  |
| 框架代理（LangChain/LlamaIndex/AutoGen 等） | 用框架封装工具注册、路由、记忆与循环执行                                    | 开发快；生态丰富；内置 agent 模式多        | 抽象层厚；调试复杂；版本升级易破坏行为    | 需要快速搭系统、愿意接受框架约束 |
| Workflow/状态机编排（代码显式控制）               | 用 DAG/状态机决定何时调用哪个工具，LLM 只做子决策                           | 可控性最高；可测试性好；合规与SLA友好         | 开发量大；灵活性较低；前期设计成本高     | 金融/医疗/企业流程、强治理场景 |
| Router + 专家模型（先路由再调用）                | 先由路由器判断任务类型，再选择工具/子agent执行                              | 成本可控；可扩展；专业任务效果更稳            | 系统复杂；路由错误会放大误差         | 多域复杂系统、流量大且任务异构  |
| 检索增强（RAG 作为“工具”）                     | 把检索系统当工具调用：检索→重排→生成                                     | 显著降低幻觉；信息可追溯；知识更新快           | 依赖索引质量；检索链路延迟与成本增加     | 知识问答、文档助手、企业内知识库 |
